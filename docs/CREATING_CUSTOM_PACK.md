## 🔍 Обзор системы паков

Warborn Renewed поддерживает **систему контент-паков** — это способ добавлять новую броню в мод без написания Java-кода. Каждый пак — это набор файлов, включающий:

- **JSON-конфигурацию** — параметры брони (защита, прочность и т.д.)
- **3D-модель** — файл `.geo.json` в формате Bedrock/GeckoLib
- **Текстуру** — файл `.png` для отображения модели
- **Анимацию** (опц.) — файл `.animation.json` для анимаций GeckoLib

Паки загружаются из папки `warbornrenewed/packs/` в корневой директории игры (рядом с папкой `mods`).

## 📁 Структура пака

Каждый пак располагается в отдельной папке внутри `packs/` в репозитории. Структура пака должна соответствовать стандарту ресурс-паков Minecraft, с добавлением папки `json` для конфигураций мода.

Вот правильная структура:

```
packs/
└── my_cool_armor/             ← Название вашего пака (snake_case)
    ├── pack.mcmeta            ← Обязательный файл метаданных
    ├── pack.json              ← Конфигурация вкладки в креативе
    ├── assets/
    │   └── my_cool_armor/     ← Namespace (обычно совпадает с именем пака)
    │       ├── geo/
    │       │   └── armor/
    │       │       └── my_helmet.geo.json     ← 3D-модель шлема
    │       ├── textures/
    │       │   └── armor/
    │       │       └── my_helmet.png          ← Текстура шлема
    │       └── models/
    │           └── item/
    │               └── my_helmet.json         ← Модель предмета в инвентаре
    ├── json/
    │   └── armor/
    │       └── my_helmet.json         ← Конфигурация характеристик шлема
    └── lang/                          ← Локализация
        ├── en_us.json
        └── ru_ru.json
```

> [!IMPORTANT]
> Имя папки пака **должно быть уникальным** и использовать формат `snake_case` (строчные буквы, слова через подчёркивание). Пример: `russian_ratnik`, `nato_urban_camo`.
> Внутри папки `assets` должна быть папка с уникальным `namespace` (обычно совпадает с именем пака).

## 🚀 Пошаговое создание пака

### Шаг 1: Создание основы пака

1. Создайте папку с уникальным именем вашего пака в директории `packs/`.
2. В корне пака создайте файл `pack.mcmeta`:

```json
{
  "pack": {
    "description": "Описание вашего пака",
    "pack_format": 15
  }
}
```

3. В корне пака создайте файл `pack.json` для настройки вкладки в креативе:

```json
{
    "tab_name": "My Custom Pack",
    "icon_item": "my_cool_armor:my_helmet"
}
```
* `icon_item` — ID предмета, который будет иконкой вкладки (формат `namespace:item_id`).

### Шаг 2: Создание ресурсов (Assets)

Создайте структуру папок для ресурсов: `assets/<namespace>/`.
Внутри `namespace` создайте папки:
- `geo/armor/` — для 3D моделей брони
- `textures/armor/` — для текстур брони
- `models/item/` — для моделей предметов (иконок в инвентаре)

#### 3D-модель (Geo)
Откройте **Blockbench**, создайте модель (Bedrock Entity/GeckoLib).

#### Иерархия костей в Blockbench

Для правильного позиционирования брони рекомендуется использовать следующую иерархию костей (как в стандартных моделях мода):

Пример иерархии для шлема:

```
📦 root
└── 🦴 bipedHead (pivot: 0, 24, 0)
    └── 🦴 armorHead (pivot: 0, 24, 0)
        ├── 🟦 helmet_base (куб)
        ├── 🟦 helmet_visor (куб)
        └── 🟦 helmet_strap (куб)
```

Пример иерархии для бронежилета:

```
📦 root
├── 🦴 bipedBody (pivot: 0, 24, 0)
│   └── 🦴 armorBody (pivot: 0, 24, 0)
│       ├── 🟦 vest_front (куб)
│       ├── 🟦 vest_back (куб)
│       └── 🟦 vest_collar (куб)
├── 🦴 bipedRightArm (pivot: -5, 22, 0)
│   └── 🦴 armorRightArm (pivot: -5, 22, 0)
│       └── 🟦 shoulder_pad_right (куб)
└── 🦴 bipedLeftArm (pivot: 5, 22, 0)
    └── 🦴 armorLeftArm (pivot: 5, 22, 0)
        └── 🟦 shoulder_pad_left (куб)
```

> [!TIP]
> Использование родительских костей `biped...` помогает избежать проблем с "сломанным центром" (смещением модели) и обеспечивает совместимость с анимациями игрока.

#### Pivot Points (точки привязки)

Для корректного расположения брони на модели игрока используйте следующие pivot point'ы:

| Кость (Bone) | Pivot Point (X, Y, Z) |
|---|---|
| `bipedHead` / `armorHead` | `[0, 24, 0]` |
| `bipedBody` / `armorBody` | `[0, 24, 0]` |
| `bipedRightArm` / `armorRightArm` | `[-5, 22, 0]` |
| `bipedLeftArm` / `armorLeftArm` | `[5, 22, 0]` |
| `bipedRightLeg` / `armorRightLeg` | `[-1.9, 12, 0]` |
| `bipedLeftLeg` / `armorLeftLeg` | `[1.9, 12, 0]` |
| `bipedRightBoot` / `armorRightBoot` | `[-1.9, 12, 0]` |
| `bipedLeftBoot` / `armorLeftBoot` | `[1.9, 12, 0]` |

3. Экспортируйте модель как **Bedrock Geometry** (`.geo.json`).
4. Сохраните файл в `assets/<namespace>/geo/armor/<name>.geo.json`.

#### Пример модели `.geo.json`

```json
{
  "format_version": "1.12.0",
  "minecraft:geometry": [
    {
      "description": {
        "identifier": "geometry.my_helmet",
        "texture_width": 64,
        "texture_height": 64,
        "visible_bounds_width": 3,
        "visible_bounds_height": 3,
        "visible_bounds_offset": [0, 1.5, 0]
      },
      "bones": [
        {
          "name": "bipedHead",
          "pivot": [0, 24, 0]
        },
        {
          "name": "armorHead",
          "parent": "bipedHead",
          "pivot": [0, 24, 0],
          "cubes": [
            {
              "origin": [-4.5, 23.5, -4.5],
              "size": [9, 9, 9],
              "uv": [0, 0]
            }
          ]
        }
      ]
    }
  ]
}
```

#### Текстура
Сохраните текстуру модели в `assets/<namespace>/textures/armor/<name>.png`.

#### Модель предмета (Item Model)
Чтобы предмет отображался в инвентаре, создайте файл `assets/<namespace>/models/item/<name>.json`:

```json
{
  "parent": "item/generated",
  "textures": {
    "layer0": "my_cool_armor:armor/my_helmet"
  }
}
```
* `layer0` указывает на путь к текстуре (относительно `assets/<namespace>/textures/`). В примере это `assets/my_cool_armor/textures/armor/my_helmet.png`.

### Шаг 3: Настройка JSON-конфигурации

Создайте файл `json/armor/<name>.json`. Этот файл связывает все ресурсы воедино и задает характеристики.

```json
{
  "id": "my_cool_armor:my_helmet",
  "type": "helmet",
  "model_id": "my_cool_armor:armor/my_helmet",
  "texture_id": "my_cool_armor:armor/my_helmet",
  "name": "My Custom Helmet",
  "names": {
    "ru_ru": "Мой крутой шлем",
    "en_us": "My Cool Helmet"
  },
  "material": "kevlar",
  "defense": 3,
  "toughness": 1.0,
  "knockback_resistance": 0.0,
  "durability": 150
}
```

- `id`: Уникальный идентификатор предмета (`namespace:item_name`).
- `model_id`: Путь к модели. `namespace:armor/name` ищет файл в `assets/namespace/geo/armor/name.geo.json`.
- `texture_id`: Путь к текстуре. `namespace:armor/name` ищет файл в `assets/namespace/textures/armor/name.png`.

### Шаг 4: Анимации (опционально)

Если вы хотите добавить анимации, создайте файл `.animation.json` рядом с моделью:

```
packs/<ваш_пак>/models/armor/<имя>.animation.json
```

Анимации создаются в **Blockbench** с использованием плагина **GeckoLib Animation Utils**:

1. Установите плагин GeckoLib в Blockbench.
2. Создайте анимации для вашей модели.
3. Экспортируйте как `.animation.json`.

Система автоматически подхватит файл анимации по пути, соответствующему `model_id` (с расширением `.animation.json`).

### Шаг 5: Локализация (имя предмета)

Есть **два способа** задать имя предмета — выбирайте тот, что удобнее:

---

#### Способ 1: Прямо в JSON конфигурации (рекомендуется ✅)

Самый простой способ — добавьте поля `name` и/или `names` прямо в JSON-файл брони:

```json
{
  "id": "my_pack:tactical_helmet",
  "model_id": "my_pack:geo/armor/tactical_helmet",
  "name": "Tactical Helmet",
  "names": {
    "ru_ru": "Тактический шлем",
    "en_us": "Tactical Helmet"
  },
  "defense": 4,
  "toughness": 1.5,
  "knockback_resistance": 0.05,
  "durability": 250
}
```

**Приоритет отображения:**
1. `names` → имя на текущем языке игры
2. `name` → общее имя (если язык не найден в `names`)
3. `id` → крайний fallback

> [!TIP]
> Если ваш пак рассчитан только на один язык — достаточно указать только `name`, без `names`.

---

#### Способ 2: Через lang-файлы в папке пака

Для продвинутых пользователей или паков с большим количеством предметов — создайте папку `lang/` внутри пака:

```
packs/my_pack/
└── lang/
    ├── en_us.json
    └── ru_ru.json
```

**Формат ключа:**
```
item.warbornrenewed.pack.<имя_пака>.<id_предмета>
```

**`lang/en_us.json`:**
```json
{
  "item.warbornrenewed.pack.my_pack.tactical_helmet": "Tactical Helmet",
  "item.warbornrenewed.pack.my_pack.tactical_vest": "Tactical Vest"
}
```

**`lang/ru_ru.json`:**
```json
{
  "item.warbornrenewed.pack.my_pack.tactical_helmet": "Тактический шлем",
  "item.warbornrenewed.pack.my_pack.tactical_vest": "Тактический жилет"
}
```

> [!NOTE]
> Lang-файлы имеют **наивысший приоритет** — если перевод найден в lang-файле, он будет использован вместо `name`/`names` из JSON.

## 📊 Справочник по параметрам брони

### Описание параметров JSON

| Параметр | Тип | Обяз. | Описание |
|---|---|---|---|
| `name` | `string` | ❌ | **Отображаемое имя** предмета. Если не указаны `names` — используется как основное имя для всех языков |
| `names` | `object` | ❌ | **Мультиязычные имена**. Ключ — локаль (`ru_ru`, `en_us` и др.), значение — имя на этом языке |
| `material` | `string` | ❌ | **Материал брони**. Определяет отображаемое название материала в тултипе. По умолчанию: `kevlar`. [Список материалов](#-список-доступных-материалов) |
| `defense` | `int` | ✅ | **Очки защиты** (аналог ванильной брони). 0–20 |
| `toughness` | `float` | ✅ | **Стойкость** брони (уменьшает повреждения от сильных атак). 0.0–4.0 |

### Список доступных материалов

Для поля `material` используйте следующие значения (в нижнем регистре):

| Значение JSON (`material`) | Отображаемое имя | Характеристики (ориентир) |
|---|---|---|
| `leather` | Leather | Слабая защита |
| `kevlar` | Kevlar | Стандартная военная броня |
| `ceramic` | Ceramic | Тяжелая защита |
| `ar500_steel` | AR500 Steel | Прочная сталь |
| `uhmwpe` | UHMWPE | Высокотехнологичный полиэтилен |
| `composite` | Composite | Композитная броня |
| `titanium` | Titanium | Максимальная защита |

> [!NOTE]
> Выбор материала влияет на отображаемое название в тултипе и базовые звуки экипировки. Сами характеристики защиты (`defense`, `toughness`) вы задаёте вручную в полях JSON.

### Рекомендуемые значения характеристик

Для ориентира — характеристики существующей брони мода:

| Тип | Предмет | Defense | Toughness | Bullet Res. | Durability |
|---|---|---|---|---|---|
| 🪖 Шлем (лёгкий) | Панама | 0 | 0.0 | 0% | Низкая |
| 🪖 Шлем (средний) | PASGT | 2 | 0.5 | 30% | 200 |
| 🪖 Шлем (тяжёлый) | 6Б47 | 3 | 1.0 | 35% | 300 |
| 🪖 Шлем (продвинутый) | Ops-Core | 3 | 1.0 | 40% | 350 |
| 🦺 Жилет (средний) | IOTV | 6 | 2.0 | 50% | 400 |
| 🦺 Жилет (тяжёлый) | 6Б45 | 7 | 2.5 | 55% | 450 |
| 🦺 Жилет (элитный) | Warmor | 8 | 3.0 | 60% | 500 |

> [!WARNING]
> Не делайте характеристики **чрезмерно высокими** — это нарушит баланс мода!

## ✅ Тестирование пака

Перед использованием протестируйте пак локально:

1. **Скопируйте пак** в папку игры:
   ```
   .minecraft/warbornrenewed/packs/<ваш_пак>/
   ```

2. **Запустите Minecraft** с установленным модом Warborn Renewed.

3. **Проверьте:**
   - ✅ Броня загружается без ошибок в логах
   - ✅ Модель корректно отображается на персонаже
   - ✅ Текстура отображается правильно (без артефактов и "розовых квадратов")
   - ✅ Характеристики (защита, прочность) работают как ожидалось
   - ✅ Название предмета отображается корректно (не как `item.warbornrenewed.pack...`)
   - ✅ Анимации работают (если используются)

4. **Проверьте логи** на наличие ошибок:
   ```
   logs/latest.log
   ```
   Ищите строки с `Failed to load armor def` или `WarbornPackManager`.

## ❓ FAQ

### Мой пак не загружается. Что делать?

1. Проверьте структуру папок — она должна соответствовать формату выше
2. Проверьте JSON на синтаксические ошибки (лишние запятые, незакрытые скобки)
3. Убедитесь, что `id` уникален и не конфликтует с другими паками
4. Посмотрите логи: `logs/latest.log` — ищите `Failed to load armor def`

### Какой формат модели нужен?

Используйте формат **Bedrock Entity Geometry** `.geo.json` (тот же, что используется для моделей Bedrock Edition и GeckoLib).

### Модель отображается неправильно (сдвинута, повёрнута)

Проверьте **pivot points** корневых костей — они должны соответствовать таблице в разделе ["Pivot Points"](#pivot-points-точки-привязки).

### Текстура отображается как розово-чёрные квадраты

Это значит, что путь `model_id` указывает на несуществующий файл текстуры. Убедитесь, что:
- `.png` файл существует рядом с `.geo.json`
- Путь в `model_id` правильный

### Могу ли я добавить несколько предметов в один пак?

Да! Создавайте по одному `.json`, `.geo.json` и `.png` файлу для каждого предмета брони, все в рамках одной папки пака.

### Какой размер текстуры лучше использовать?

Рекомендуется **64×64** для простых моделей и **128×128** или **256×256** для более детальных. Текстуры должны быть квадратными и размером, кратным степени двойки (64, 128, 256). Размер текстуры **должен совпадать** с указанным в `.geo.json` (`texture_width` / `texture_height`).
